# Google Analyticsアクセス解析の設定手順

## 1. Google Analytics 4 のセットアップ

### A. Google Analyticsアカウント作成
1. [Google Analytics](https://analytics.google.com/) にアクセス
2. 「測定を開始」をクリック
3. アカウント名を入力（例：動画編集資金計算ツール）
4. データ共有設定を選択（推奨設定のまま）

### B. プロパティ作成
1. プロパティ名を入力（例：Video Cash Flow Calculator）
2. レポートのタイムゾーン：日本
3. 通貨：日本円（JPY）

### C. データストリーム設定
1. 「ウェブ」を選択
2. ウェブサイトのURL（例：https://your-domain.com）
3. ストリーム名を入力
4. **測定ID (G-XXXXXXXXXX)** をコピー

## 2. Next.jsアプリでの設定

### A. 環境変数設定
`.env.local` ファイルを作成（gitignoreに含まれている）:

```bash
# Google Analytics 測定ID
NEXT_PUBLIC_GA_ID=G-XXXXXXXXXX
```

### B. 開発環境での動作確認
```bash
npm run dev
```

開発中は、ブラウザの開発者ツール（Console）で以下を確認:
- `gtag` 関数が利用可能
- エラーメッセージがないこと

## 3. 確認できる分析データ

### 基本データ
- **ユーザー数**: 訪問したユニークユーザー数
- **ページビュー**: 閲覧されたページ数
- **セッション**: 訪問回数
- **直帰率**: 1ページのみ閲覧して離脱した割合

### 詳細分析
- **地域**: 訪問者の地理的分布
- **デバイス**: モバイル/デスクトップ/タブレットの比率
- **ブラウザ**: Chrome/Safari/Firefox等の使用状況
- **リアルタイム**: 現在のアクティブユーザー数

### カスタムイベント（本アプリで設定済み）
- **cash_flow_calculation**: 資金計算の実行回数
- **form_submit**: フォーム送信数
- **button_click**: ボタンクリック数
- **page_view**: ページビュー詳細

## 4. データの確認方法

### A. Google Analyticsダッシュボード
1. [Google Analytics](https://analytics.google.com/) にログイン
2. 作成したプロパティを選択
3. レポート > リアルタイム で即座にデータ確認
4. レポート > エンゲージメント で詳細分析

### B. 主要レポート
- **リアルタイム**: 現在のアクセス状況
- **エンゲージメント > ページとスクリーン**: ページ別アクセス数
- **エンゲージメント > イベント**: カスタムイベントの分析
- **ユーザー属性 > 地域**: 地理的分析
- **テクノロジー > ユーザーの環境**: デバイス・ブラウザ分析

## 6. トラブルシューティング

### データが表示されない場合
1. 測定IDが正しく設定されているか確認
2. `.env.local`ファイルがgitignoreされていることを確認
3. ブラウザの開発者ツールでエラーを確認
4. Google Analyticsの設定で、自分のIPアドレスが除外されていないか確認

### 本番環境でのチェックポイント
1. 環境変数 `NEXT_PUBLIC_GA_ID` が設定されていることを確認
2. ドメイン設定がGoogle Analyticsと一致していることを確認
3. HTTPS接続が有効になっていることを確認
